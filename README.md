# PassPilot — מנווט המעבר 🎯

כלי למידה לסטודנט שצריך **רק לעבור** מבחנים (ציון 50-61), לא לדעת 60% מהחומר.
העיקרון: **לבנות יכולת להוציא 55 נקודות אמין** — לא לכסות את כל החומר.

> נבנה מרעיון של אבירן ("תוכנית למידה אופטימלית"), מבוסס מחקר: שליפה מהזיכרון (active recall) ולמידה מרווחת יעילות משמעותית מקריאה חוזרת/צפייה פסיבית.

## הרעיון: לוח בקרת מעבר (Pass-Control-Board)
במקום לתכנן "נושאים ושעות", מתכננים לפי **מרחק מהמעבר**. לכל קורס מזינים את הציון שמוציאים **עכשיו במבחן-עבר מתוזמן**, והלוח מחליט על מה לעבוד:

| ציון סימולציה | פעולה |
|---|---|
| 45–59 | ⚡ **עדיפות עליונה** — הכי קרוב למעבר, הכי משתלם |
| 30–44 | נושאים **זולים בנקודות** |
| 50–61 עקבי | תחזוקה בלבד ✓ |
| מתחת ל-25 | רק יסודות ונושאים בטוחים — לא לנסות לכסות הכול |

**חלוקת זמן יומית:** 65% למבחן הקרוב · 25% לבא אחריו · 10% לשחזור טעויות. עד 4 ימים למבחן → 80/15/5.

## עקרונות הליבה (המתודולוגיה)
1. **תפתור, אל תצפה** — active recall ומבחני-עבר במקום וידאו פסיבי.
2. **מבחני-עבר קודם** — אבל נתקעת 5-8 דק'? למד פתרון מלא אחד, סגור, שחזר לבד, פתור דומה. לא להיאבק 30 דק' בלי בסיס.
3. **מפה נקודות, לא שכיחות** — נושא שחוזר אבל קשה מאוד = השקעה גרועה מול 3 נושאים קלים של 10 נק'.
4. **רשימת "שאלות שאני לוקח / גיבוי / מוותר"** לכל קורס.
5. **פומודורו = משמעת, לא שיטה** — 35-50 דק' אם עובד; הראיות לשליפה חזקות בהרבה.
6. **AI/NotebookLM רק** להסביר טעות / לייצר וריאציה / לבחון — לא לבנות כרטיסיות יפות במקום לפתור.

## MVP (בריפו הזה)
`index.html` — לוח-בקרת-מעבר עובד: מוסיפים קורסים + תאריך מבחן + ציון סימולציה, מקבלים עדיפות-פעולה לכל קורס וחלוקת-זמן יומית. נתונים נשמרים מקומית (localStorage). אפס תלויות, עובד ב-GitHub Pages.

## הפעלה
פתח את `index.html` בדפדפן, או דרך GitHub Pages.
